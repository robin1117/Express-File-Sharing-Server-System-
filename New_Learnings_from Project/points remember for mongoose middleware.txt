pre('save') middleware runs before a document is saved.
In pre('save'), this refers to the current document.
Async middleware should not use next().
Callback-style middleware uses next() to continue execution.

Passwords should never be stored in plain text.
bcrypt.hash() converts a password into a secure hash.
Bcrypt generates a random salt for every hash.
The same password produces a different hash each time.
The salt is stored inside the hash itself.
The original password cannot be recovered from the hash.
bcrypt.compare() is used to verify a password.
Password verification should not be done using === between hashes.

Middleware should use regular functions (function () {}), not arrow functions.
Arrow functions do not provide the correct this context in middleware.
